# Configuring the installation



ImgArchive has a flexible method of installation in order to maximize the available hardware. In order to realise this flexibility ImgArchive has a range of installation configuration options. This section details these options and how best to use them.

Configuring the installation consists of taking the standard installation and describes how to tailor this installation and re configure it to maximize it to your requirements.

These areas possibly need changes in configuration:

1. General configuration such as logging level etc.
2. The Vault.
3. The User space.

## Configuring the import work flow

 

 

ImgArchive installs into a default setup which is located in three areas on the home disk normally drive C: these areas are as follows:

- Binary folder - C:\Program Files\IDK Software\ImgArchive
- Configuration folder - C:\ProgramData\IDK Software\ImgArchive
- Working area folder - C:\Users\<Your User Name>\ImgArchive

These folders contains the following:

- **Binary folder** - Contains the ImgArchive binaries
- **Configuration folder -** Contains the configuration including default configuration files
- **Working area folder -** This area is used by the user. The working image file are contained in this area.

These folders may need to be optimised to maximise drive usage.  These default locations are using only the home disk drive c: It is more than likely this drive is not large enough to support the Image archive and the backup areas are disabled in the default installation.
