# Configuring the import work flow

When importing images into the archive a number of features can be setup to enhance the importing of images into the archive. These are as follows:

- File naming
- Exif reading
- Default metadata template

## File naming

File naming is the renaming of each image file to be unique within the archive. When images are imported into the archive image file names may be the same. If the image file has the same file name as a previously imported image it will over-write the previous one. ImgArchive will take steps to prevent this from happening. For a full explanation why this is important see [File naming](https://idk-solutions.atlassian.net/wiki/spaces/ImgaWiki/pages/1731624999).

## Exif reading

ImgArchive contains an internal Exif reader however it is limited to picture images like jpg which is fine if you only shoot jpg (many do). However if you shoot RAW images then installing an external reader into ImgArchive may be a good idea. If this is a option you would like to pursue then see [Using an external Exif reader](https://idk-solutions.atlassian.net/wiki/spaces/ImgaWiki/pages/1729396797).  

## Default metadata template

Then importing a set of images that have similar attributes then to save these on bulk will save a lot of time than to add these attributes to each image.
